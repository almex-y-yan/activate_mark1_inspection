# COM設定 + サービス再起動ツール 操作マニュアル

本書は、キオスク PC 上で `web.exe` を使い、画面（ブラウザ UI）から操作する手順を説明します。

---

## 1. このツールでできること

| 操作 | 内容 |
|------|------|
| **出荷検査ツール開始** | 出荷検査用にサービスを停止・開始し、`mark1_inspection.exe`（自己診断ツール）を起動する |
| **COM設定+サービス再起動実行** | Card / IRS / NM43 の COM 番号を `.ini` に書き込み、対応サービスを再起動する |
| **再読込** | 現在の `.ini` の COM 設定を画面に表示し直す |

---

## 2. 実行前の前提

次の条件を満たしていること。

- **Windows** が動作していること
- 環境変数 **`ALMEXPATH`** が設定されていること
- `ALMEXPATH\ini` に次のファイルがあること  
  - `almex_card_crl31.ini`  
  - `almex_iccard_irs270.ini`  
  - `almex_iccard_nm43.ini`
- ツール配置フォルダに次があること  
  - `web.exe`  
  - `web` フォルダ（`index.html` / `app.js`）

---

## 3. 管理者権限で実行すること（必須）

サービスの停止・開始および `.ini` の更新には、**管理者権限**が必要です。  
**必ず「管理者として実行」で `web.exe` を起動してください。**

### 3.1 ショートカットの作り方（推奨）

1. エクスプローラーで `web.exe` があるフォルダを開く
2. `web.exe` を右クリック → **ショートカットの作成**
3. 作成したショートカットを右クリック → **プロパティ**
4. **ショートカット** タブで次を確認・設定する  

   | 項目 | 設定例 |
   |------|--------|
   | **リンク先** | `D:\almex\tool\自己診断起動ツール\web.exe`（実際のパス） |
   | **作業フォルダ** | `web.exe` と **同じフォルダ**（例: `D:\almex\tool\自己診断起動ツール`） |

   > **作業フォルダ** は `web.exe` と同じ場所にしてください。別のフォルダにすると UI が正しく表示されない場合があります。

5. **詳細設定** を開く → **管理者として実行** にチェック → OK
6. ショートカットをタスクバーなどにピン留めする

### 3.2 起動の確認

- ショートカットをダブルクリックしたとき、**ユーザーアカウント制御（UAC）** で許可を求められたら **はい** を選ぶ
- 数秒以内にブラウザが開き、画面タイトル付近に **COM設定 + サービス再起動** が表示されれば起動成功
- 自動で開かない場合は、ブラウザで次の URL を開く  

  `http://127.0.0.1:18080`

### 3.3 管理者以外で起動した場合

次のようなエラーになることがあります。

- サービス停止・開始の失敗（アクセス拒否など）
- `.ini` の保存失敗

その場合は **一度 `web.exe` を終了**し、**3.1 のショートカット**からやり直してください。

---

## 4. 画面の見方

起動後、ブラウザに次の構成の画面が表示されます。

```
自己診断ツール（Mark1_inspection.exe）起動するサービスの設定
COM設定 + サービス再起動

[説明ボックス] … 出荷検査開始時に固定起動するサービスなどの案内

[ステータス行] … 処理結果（緑=成功、赤=失敗、青=処理中）

Card   … Card リーダーの COM 設定
IRS    … IRS リーダーの COM 設定（DEVICE1 / DEVICE2）
NM43   … NM43 リーダーの COM 設定

[出荷検査ツール開始]
[COM設定+サービス再起動実行]
[再読込]

ログ … 処理の詳細メッセージ
```

各フィールドセットの **「現在値: …」** には、`.ini` から読み取った COM 番号が表示されます。

## 5. 出荷検査ツールを開始する手順

出荷検査（自己診断）を行うときの標準手順です。

### 5.1 事前に COM を合わせる（必要な場合）

1. 画面の **Card / IRS / NM43** で、対象にするデバイスにチェックを入れる
2. 各 **Com** 番号を入力する
3. **COM設定+サービス再起動実行** を押し、成功するまで待つ（後述 **6.**）

### 5.2 出荷検査の開始

1. **Card / IRS / NM43** の **「対象にする」** を、検査で使うデバイスだけ ON にする  
   - チェック ON … 出荷検査開始時にそのサービスを **開始** する  
   - チェック OFF … そのサービスは **開始しない**（停止処理は行う）
2. **出荷検査ツール開始** をクリックする
3. ステータスが **処理中（青）** から **完了（緑）** になるまで待つ（最大約 5 分）
4. **ログ** に次のような行が出ていれば、一連の処理が進んでいる  
   - `停止: …` / `開始: …`  
   - `待機: almdevic5 起動安定化のため 2秒`（NM43 開始成功時）  
   - `起動: d:\almex\tool\mark1_inspection\mark1_inspection.exe -resetamount`
5. 自己診断ツール（`mark1_inspection.exe`）の画面が起動することを確認する

### 5.3 固定で開始されるサービス

**出荷検査ツール開始** では、次のデバイスサービスは画面のチェックに関係なく **常に開始** されます。

- Cashless PAX A35/GMO、CashMachine AD-XR、MIX Board、プリンタ、QR、Sound、Camera、HID など（画面上部の説明ボックス参照）

**Card / IRS / NM43** の 3 つだけが、**「対象にする」** のチェックで開始の有無が変わります。

### 5.4 一部サービスが失敗した場合

ログに `サービス失敗あり` と出ても、可能な範囲で **自己診断ツールの起動は続行** されます。  
検査に必要なデバイスが起動しているか、ログと実機を確認してください。

## 6. COM 設定を変更してサービスを再起動する手順

.ini の COM 番号を変更するときの手順です。

### 6.1 入力ルール

| 項目 | ルール |
|------|--------|
| **対象にする** | ON にした対象だけ `.ini` を更新する |
| **Card Com** | Card を対象にする場合は **必須**（1 以上の整数） |
| **IRS … DEVICE1 Com** | IRS を対象にする場合は **必須** |
| **DEVICE2 を使用** | ON … DEVICE2 セクションを有効化して COM を更新 / OFF … DEVICE2 セクションをコメントアウト |
| **DEVICE2 Com** | DEVICE2 を使用する場合は **必須** |
| **Nm43 Com** | NM43 を対象にする場合は **必須** |

### 6.2 操作手順

1. 必要に応じて **再読込** で現在値を確認する
2. **対象にする** と **Com** 番号を入力する
3. **COM設定+サービス再起動実行** をクリックする
4. ステータスが成功（緑）になることを確認する
5. **ログ** で次を確認する  
   - `バックアップ作成: …`（変更があった場合）  
   - `サービス停止: …` → `ini 保存完了` → `サービス開始: …` → `完了`

変更がなかった場合はログに **変更なし** と表示され、サービス再起動は行われません。

### 6.3 失敗した場合

- ログとステータス（赤）のメッセージを確認する
- 権限エラーの場合は **管理者として実行** しているか確認する
- `.ini` の復旧が必要な場合は、`ALMEXPATH\ini\backup` 内の `.bak` から元ファイルへ戻し、該当サービスを再起動する（詳細は社内復旧手順）

---

## 7. 再読込

- **再読込** を押すと、`ALMEXPATH\ini` から COM 設定を読み直し、画面の **現在値** を更新します
- 他の操作を行う前に、現在の設定を確認したいときに使います

---

## 8. ログについて

| 種類 | 場所 |
|------|------|
| 画面の **ログ** エリア | 直近の操作結果（ブラウザ上） |
| サーバーログファイル | `%ALMEXPATH%\log\ini-web-tool.log` |

トラブル調査時は、画面のログと上記ファイルの **日時が近い行** を合わせて確認してください。

## 9. よくある質問

### Q. ブラウザが開かない

- 手動で `http://127.0.0.1:18080` を開く
- 別の `web.exe` が既に動いていないか確認する（タスクマネージャー）

### Q. 「ALMEXPATH が未定義です」などで起動しない

- システムの環境変数 `ALMEXPATH` が設定されているか、管理者に確認する

### Q. キオスクにコピーするファイルは？

次だけを **同じフォルダ** に置きます。

- `web.exe`
- `web` フォルダ
- `scripts` フォルダ（初回セットアップ用）
- `bin` フォルダ（運用で必要な場合）
